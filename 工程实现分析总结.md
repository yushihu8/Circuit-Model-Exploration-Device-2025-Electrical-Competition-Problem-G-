# 2025_G 工程实现分析总结

## 1. 工程总体目标

这套工程本质上是在做一个“板上自举式系统辨识 + 数字复现”系统。

它的核心目标不是单纯做一个固定滤波器，而是：

1. 在没有外部测试设备的前提下，对未知模拟电路进行自动测试。
2. 获取该电路在一段频率范围内的频率响应。
3. 根据频率响应建立一个二阶模拟模型。
4. 再将这个模拟模型转换为 FPGA 中可实时运行的数字 IIR 滤波器。

也就是说，这套系统实现的是“自主学习、建模、再复现”。

## 4. 为什么它可以不依赖外部测试设备

传统测试一个未知模拟电路，通常需要：

1. 函数信号发生器。
2. 示波器或频响分析仪。
3. 人工记录输入输出关系。

而这套系统把这些功能全部集成到了板上：

1. `AD9954` 充当扫频信号源。
2. 双路 ADC 同时采集参考输入和电路输出。
3. FPGA 负责高速缓存采样数据。
4. STM32 完成 FFT 和传递函数计算。
5. STM32 进一步完成模型辨识和数字化。

因此它本质上是一个简化版的“板载频响测试与建模系统”。

## 6. 扫频与自主学习过程

扫频函数位于：

[freq_scan.c](C:/Users/YuDengxiang/Desktop/Electronic%20Design%20Competition/2025_G/G_Project-32/G_Project-32/Middleware/freq_scan.c)

### 6.1 扫频范围

当前代码设定：

1. 起始频率：200 Hz
2. 步进：200 Hz
3. 点数：250 点
4. 范围约为：200 Hz 到 50 kHz

### 6.2 每个频点的动作

在每个频率点，系统按如下顺序工作：

1. STM32 控制 AD9954 输出当前频率的正弦信号。
2. 等待模拟电路稳定。
3. STM32 向 FPGA 写寄存器 `0xBB01`，触发一次采样。
4. FPGA 将双通道采样数据写入 RAM。
5. STM32 通过 SPI 从 RAM 中把 1024 组双通道数据读回来。
6. 对输入通道做 FFT。
7. 对输出通道做 FFT。
8. 取当前频率对应频谱点上的复数值。
9. 计算：

```text
H(jw) = 输出复数频谱 / 输入复数频谱
```

10. 保存该频率点的复频响数据。
11. 将 DDS 切换到下一个频点。

这样循环 250 次后，就得到了整个被测电路的频率响应曲线。

### 6.3 这就是“自主学习”

因为系统不是靠预先知道电路参数来设计滤波器，而是先自己发激励、自己采集、自己分析，再自己算出模型，所以这一步就是整个系统的“自主学习”阶段。

## 7. FFT 分析方式

FFT 处理在：

[fft.c](C:/Users/YuDengxiang/Desktop/Electronic%20Design%20Competition/2025_G/G_Project-32/G_Project-32/Hardware/fft.c)

关键点如下：

1. 一次采样长度是 `1024` 点。
2. 双通道数据按交错方式存放在 `adc_buff` 中。
3. `fft_calc_a()` 处理输入参考通道。
4. `fft_calc_b()` 处理输出通道。
5. 两次 FFT 后，在对应 bin 上提取实部和虚部。
6. 通过复数除法得到当前频率点的传递函数。

这意味着系统保留的不是单纯幅值，而是完整的复频响信息，包括：

1. 幅频特性。
2. 相频特性。

因此后续才能建立动态模型，而不仅仅是估计增益。

## 8. 建模方法

建模核心位于：

[h_calc.c](C:/Users/YuDengxiang/Desktop/Electronic%20Design%20Competition/2025_G/G_Project-32/G_Project-32/Hardware/h_calc.c)

### 8.1 建模目标

程序试图用一个二阶模拟滤波器来逼近未知电路：

```text
H(s) = (b2*s^2 + b1*s + b0) / (a2*s^2 + a1*s + 1)
```

也就是说，系统默认被测电路可以近似为一个二阶系统。

### 8.2 系数求解方法

程序用扫频得到的 250 个复频响点，构造一个最小二乘意义下的线性方程组，然后解出：

1. `b0`
2. `b1`
3. `b2`
4. `a1`
5. `a2`

这一步的意义是：

把“测得的频域响应”变成“一个明确的数学模型”。

### 8.3 滤波器类型识别

程序还通过比较：

1. `|H(0)|`
2. `|H(∞)|`

自动判断被测对象更像：

1. 低通
2. 高通
3. 带通
4. 带阻

这说明这套系统不仅在拟合参数，也在对电路功能做自动分类。

## 9. 从模拟模型到数字模型

得到模拟滤波器后，程序继续通过双线性变换，把它转成 FPGA 能直接跑的数字 IIR。

目标数字滤波器形式为：

```text
H(z) = (b0 + b1*z^-1 + b2*z^-2) / (1 + a1*z^-1 + a2*z^-2)
```

随后再把这些系数量化成定点格式，并写入 FPGA。

这一步完成后，系统就把“模拟电路特性”转成了“数字滤波器特性”。

## 10. FPGA 的作用

FPGA 顶层文件：

[top.v](C:/Users/YuDengxiang/Desktop/Electronic%20Design%20Competition/2025_G/G_project/G_project/G_Project.srcs/sources_1/new/top.v)

### 10.1 ADC 采样模块 `ad9248`

作用：

1. 由 FPGA 输出 ADC 时钟。
2. 在设定节拍下锁存 ADC 数据。
3. 输出 `adc_val` 表示当前采样有效。

其中：

1. `adc_in0` 对应一路参考/输入采样。
2. `adc_in1` 对应一路输出采样，同时也作为 IIR 的输入。

### 10.2 RAM 采样缓存模块 `adc_ram_control`

作用：

1. 接收双通道 ADC 数据。
2. 当收到 `adc_ram_en` 后开始采样存储。
3. 连续写入 1024 组数据到 RAM。
4. 每组数据为 28 位，由两路 14 位 ADC 拼成。

STM32 后续通过 SPI 再从 RAM 中读回这些数据。

### 10.3 SPI 控制模块 `spi_reg_control`

作用：

1. 接收 STM32 发来的 SPI 命令。
2. 做寄存器读写解析。
3. 提供 RAM 读写接口。
4. 保存 IIR 系数和控制参数。

### 10.4 IIR 模块 `iir_filter`

文件：

[iir_filter.v](C:/Users/YuDengxiang/Desktop/Electronic%20Design%20Competition/2025_G/G_project/G_project/G_Project.srcs/sources_1/new/iir_filter.v)

作用：

1. 使用 STM32 下发的二阶 IIR 系数。
2. 按差分方程进行实时滤波。
3. 将滤波结果送往 DAC。

因此 FPGA 的作用不是建模，而是：

1. 协助高速采样。
2. 提供数据缓存。
3. 执行最终实时数字滤波。

## 11. SPI 寄存器映射关系

寄存器控制位于：

[reg_control.v](C:/Users/YuDengxiang/Desktop/Electronic%20Design%20Competition/2025_G/G_project/G_project/G_Project.srcs/sources_1/new/reg_control.v)

当前关键地址如下：

### 11.1 采样触发寄存器

- `0xBB01`

作用：

STM32 写入该地址后，FPGA 启动一次 ADC 采样并将结果写入 RAM。

### 11.2 系数寄存器

- `0xA001 ~ 0xA003`：`a1`
- `0xA004 ~ 0xA006`：`a2`
- `0xA007 ~ 0xA009`：`b0`
- `0xA00A ~ 0xA00C`：`b1`
- `0xA00D ~ 0xA00F`：`b2`

每个系数是 48 位，被拆成 3 个 16 位字通过 SPI 写入。

### 11.3 采样速率寄存器

- `0xA010`

作用：

控制 `adc_in1` 这一路 ADC 的采样时钟分频参数。

### 11.4 RAM 地址空间

- `0x0000 ~ 0x07FF`

作用：

STM32 通过 SPI 从这些地址读取采样缓存内容。

## 14. 一个很重要的理解

这套系统实际上有两个工作阶段。

### 14.1 第一阶段：辨识阶段

目标：测未知电路。

特点：

1. AD9954 逐点扫频。
2. FPGA 采集参考输入和输出。
3. STM32 做 FFT 和频响计算。
4. STM32 建立二阶模型。

这一阶段本质上是在“学”。

### 14.2 第二阶段：运行阶段

目标：用数字方式复现它。

特点：

1. STM32 已经算出了数字 IIR 系数。
2. 系数被下发到 FPGA。
3. FPGA 内部 IIR 开始实时工作。
4. DAC 输出数字滤波后的结果。

这一阶段本质上是在“用学到的模型工作”。

所以可以把它概括成：

**先学习，再建模，再复现。**

从 `main.c` 实际调用关系看，当前真正启用的是这条链路：

1. `AD9954` 激励。

4. `FPGA IIR` 数字实现。

虽然工程里还保留了 `MAX262` 和 `PWM_Set()` 相关代码，但这些代码在当前主流程中并没有被实际调用。

因此可以认为：

1. `MAX262` 相关部分更像历史方案或备用方案。
2. 当前真正工作的方案是“数字建模 + FPGA 数字滤波复现”。

## 17. 最终总结

这两个工程组合起来，完成的是一个完整的板上闭环系统：

1. `AD9954` 负责提供激励。
2. 模拟电路作为被测对象。
3. FPGA 负责采样、缓存和实时数字滤波执行。
4. STM32 负责扫频分析、频响计算、模型辨识和系数下发。

因此这套系统不是简单地“控制一个 FPGA 或控制一个滤波器”，而是：

**自动测量未知模拟电路 → 自动建立模型 → 自动把模型转成数字滤波器 → 在 FPGA 内实时运行。**

这正是它实现“自主学习、建模，且不可借助外部测试设备”的本质。

## 18. 额外说明

从代码实现上看，主思路是正确且完整的，但当前工程中仍有少量实现细节值得后续复核，例如：

1. `coef_lode_en` 在顶层连接中没有真正作为加载脉冲使用。
2. `coef_lode()` 中对 `0xA010` 的写入长度看起来与寄存器含义不完全一致。
3. 部分注释中对采样率的表述与 FPGA 默认扫频阶段的实际分频值存在不完全一致。

这些问题不影响对整套系统总体结构和工作原理的理解，但如果后续要用于答辩、复现或修改代码，建议再做一次针对实现细节的核对。
